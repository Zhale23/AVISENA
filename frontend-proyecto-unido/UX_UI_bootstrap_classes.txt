Guía de clases Bootstrap 

Notas generales:
- Preferir utilidades Bootstrap en lugar de CSS personalizado cuando sea posible.

**Estructura de Tablas**:
- Contenedor responsivo: envolver la tabla en `<div class="table-responsive">`.
- Tabla base: `table table-striped table-hover table-sm mb-0 text-nowrap align-middle`.
- Encabezados: usar `th` con utilidades: `fw-semibold` (peso seminegro), `text-muted` si se desea tono gris.
- Celdas de acciones: añadir `text-end` al `th` o `td` para alinear a la derecha.
- Filas con acciones: en `td` usar `d-flex justify-content-end gap-2` y botones `btn-sm`.
- Ejemplo:
  <div class="table-responsive">
    <table class="table table-striped table-hover table-sm mb-0 text-nowrap align-middle">
      <thead>
        <tr>
          <th class="fw-semibold">Nombre</th>
          <th class="fw-semibold text-end">Acciones</th>
        </tr>
      </thead>
      <tbody id="items-table-body"></tbody>
    </table>
  </div>

**Estructura botón "Agregar"**:
- Clases recomendadas: `btn btn-success` (usar `btn-sm` si el diseño requiere botones compactos).
- Icono + texto: `<button class="btn btn-success"><i class="fa fa-plus me-1"></i>Agregar</button>`
- Cuando esté en el header de una card: agrupar controles con `d-flex align-items-center gap-2` o `btn-group`.

**Estructura botón "Editar"**:
- Clases: `btn btn-success btn-sm` (verde, consistente con acción positiva de edición en este proyecto).
- Icono: `<button class="btn btn-success btn-sm" aria-label="Editar"><i class="fa fa-pen me-0"></i></button>`
- Preferir solo icono en tablas (espacio reducido). Añadir `title="Editar"` o `aria-label`.

**Estructura botón "Eliminar"**:
- Clases: `btn btn-secondary btn-sm` (gris/neutral en este proyecto). Si prefieres más contraste, `btn-outline-secondary btn-sm`.
- Icono: `<button class="btn btn-secondary btn-sm" aria-label="Eliminar"><i class="fa fa-trash me-0"></i></button>`
- Confirmación: siempre usar modal o SweetAlert2 para evitar borrado accidental.

**Iconos (ejemplos)**:
- Agregar: `fa fa-plus` (o `fa-solid fa-plus` si usan FontAwesome 6)
- Editar: `fa fa-pen` / `fa fa-edit`
- Eliminar: `fa fa-trash`
- Exportar: `fa fa-file-export` o `fa fa-file-download`
- Usuarios: `fa fa-users`
- Espaciar iconos: usar `me-1` (margin-end) para separar icono y texto.

**Estructura de Switch (toggle)**:
- Usar Bootstrap native switch:
  <div class="form-check form-switch">
    <input class="form-check-input" type="checkbox" id="switch-{{id}}" data-id="{{id}}">
    <label class="form-check-label" for="switch-{{id}}">Activo</label>
  </div>
- En JS, adjuntar listener `change` al `input.form-check-input`. Ejecutar confirmación (SweetAlert2) y revertir el `checked` si el usuario cancela o si la petición falla.
- Mantener atributo `data-id` y `data-*` para identificación en peticiones API.

**Estructura de `select` / listas desplegables**:
- Clases base: `form-select`.
- Mejor apariencia (proyecto): `form-select form-select-sm rounded border-secondary shadow-sm bg-white`.
  - `form-select-sm`: tamaño compacto
  - `rounded`: esquinas suaves
  - `border-secondary`: borde gris (en vez del azul por defecto)
  - `shadow-sm`: sutil elevación
  - `bg-white`: fondo blanco uniforme
- Si se necesita ancho automático: combinar con `w-auto` o `style="width:auto;"` y utilidades `me-2` para separación.
- Notas: el color del highlight interno de opciones puede depender del SO; para consistencia total usar un select custom (Choices.js). Se agregó `assets/css/select-fix.css` con una pequeña regla de foco si no se usa una librería.
- Ejemplo:
  <select id="filter-estado" class="form-select form-select-sm rounded border-secondary shadow-sm bg-white me-2" style="width:auto;">
    <option value="all">Todos</option>
  </select>

**Modales (crear / editar)**:
- Estructura habitual:
  <div class="modal fade" id="edit-modal" tabindex="-1" aria-hidden="true">
    <div class="modal-dialog">
      <div class="modal-content">
        <div class="modal-header">
          <h5 class="modal-title">Editar</h5>
          <button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button>
        </div>
        <div class="modal-body"> ... </div>
        <div class="modal-footer">
          <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">Cancelar</button>
          <button type="submit" class="btn btn-success">Guardar Cambios</button>
        </div>
      </div>
    </div>
  </div>
- Botones: `Cancelar` => `btn btn-secondary`, `Acción` => `btn btn-success`.

**Inputs y formularios**:
- Entradas: `input.form-control`, `textarea.form-control`.
- Wrapper: `div.mb-3` alrededor de cada label+input.
- Labels: `label.form-label` y `for` apuntando al `id` del input.
- Validaciones: usar `required` en markup cuando aplique.

**Paginación**:
- Estructura: `<nav><ul class="pagination justify-content-center">...` con `li.page-item` y `a.page-link`.
- Compacto: usa utilidades `pagination-sm` si se necesita más compacto.

**Export / Dropdown**:
- Botón de exportar: agrupar en `btn-group`:
  <div class="btn-group">
    <button class="btn btn-outline-secondary dropdown-toggle" data-bs-toggle="dropdown">Exportar</button>
    <ul class="dropdown-menu">...
    </ul>
  </div>
- Usar `btn-outline-secondary` para acciones menos prioritarias.

**Acciones en celdas (recomendado)**:
- Agrupar en `div.d-flex justify-content-end gap-2` y usar `btn-sm`:
  <td class="text-end">
    <div class="d-flex justify-content-end gap-2">
      <button class="btn btn-success btn-sm" title="Editar"><i class="fa fa-pen"></i></button>
      <button class="btn btn-secondary btn-sm" title="Eliminar"><i class="fa fa-trash"></i></button>
    </div>
  </td>

**Clases utilitarias comunes que usamos**:
- Espaciado: `me-1`, `me-2`, `gap-2`, `mb-3`, `mt-3`, `mb-0`
- Texto: `text-muted`, `fw-semibold`, `text-end`, `text-center`
- Display / layout: `d-flex`, `align-items-center`, `justify-content-between`, `ms-auto`, `w-auto`
- Color de botón: `btn-success` (acciones principales), `btn-secondary` (acciones secundarias)

**JS / Integración**:
- Para switches, mantener `data-id` en el `input` para identificar el registro.

**Accesibilidad**:
- Botones solo-icono => añadir `aria-label` y `title`.
- Inputs y selects => `label` ligado por `for` y `id`.
- Contraste: el color `btn-success` (#198754) tiene buen contraste para textos blancos en botones.
